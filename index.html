Prototype for prediction of forest fires.

This project arises as a collaboration of Grupo Plenum, Texas A&M University, and the SIIES of Yucatan, to help in the detection and prediction of fires, as well as the detection of the spread of fire, using temperature and humidity sensors, and subsequently to the collection of Big Data, be able to generate mathematical models that help to achieve these objectives.

The project consisted in design an embedded system based on the microcontroller PSOC 4 of Cypress, a sensor of temperature and humidity, a Wifi module, a Bluetooth module, and a lipo battery charger module. The PCB was designed using the electronic design software EAGLE. The assembly of all the components was handcrafted. After assembly, the programming of the microcontroller to interact with all the modules was done using C and the commands referenced in the datasheets of each module.

The prototype was successful, it even had a second version making it smaller and using a better sensor, and the final functioning of it was really good. However, the project did not come to fruition due to lack of funds, and it was finished.

Merge Rack.

This project arises with the necessity of having better control in a process in a sub-assembly area, due to before of it the control was made in paper sheets and during the change shifts of the operators, normally lost information causing delays in the production.

The first MVP was made using C# to make windows forms, linked with Access as the database. After the success of the first implementation, more features were requested and finally were linked with Microsoft SQL Server. That allowed implementing the same solution with more sub-assemblies from other areas, and have all the information concentrated in the same database that was located in the main server, having the benefit of being able to consult the information from anywhere.

Maintenance Reports System.

This project arises with the necessity of register the maintenance services requested to the different equipment in the areas around the plant, to know the efficiency of the technician in charge of that equipment. With the system, it was intended to know the time when was reported the service, the response time, the solution time, and the confirmation time from the requester. The quality, production, and maintenance departments were involved in this process, due to the responsibilities of each one regarding the normal flow of the process.

The project was developed using C # to make Windows forms and Microsoft SQL Server as a database. I developed the ER-Model, all the classes needed for the CRUD, and the interface. Due to the database was located on the main server, was able to consult the information from anywhere on the plant.

Another colleague developed a web page linked to the database that displayed on TVs over the equipment in different areas to be easy to see when a new service was requested.
